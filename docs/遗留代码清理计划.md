## 当前主链（清理时勿动）

```
运行期（100 家批跑）:  pipeline.run_field → route_field(routed?) → 冷启动 → L1/L2/L3 自愈
构建期（改解析器）:    batch_runner(heal=True) → auto_heal → code_generator.repair → certify
消费复利:             route_field 读 parser_catalog / route_cache → 下次 routed 免 LLM
人工 codegen:         控制台 Review → recode → certify_parser
```

**codegen 已接主链**（两条轨，不要和 `run_full_pass` 混为一谈）：

| 轨 | 状态 | 说明 |
|----|------|------|
| **产物消费** | ✅ | `run_field` 第一步 `route_field` 跑 `parsers/versions/*.py` 认证解析器（100 家里 routed 52 家即此） |
| **构建闭环** | ✅ | `batch_runner` + `auto_heal_field` → 抽 golden → `repair`/`generate_parser` → `certify` 入目录 |
| **运行期 LLM 链内自动 codegen** | ❌ | `run_full_pass` 链尽后仍走 `judge_diagnose` 交人工；`rule_code_diagnose` 仅 JudgeTest 手动二阶段 |

因此 **`code_generator` / `auto_heal` 不是废弃代码**；缺的是「非绿灯 diagnose 后自动 fork 写解析器」，不是 codegen 能力本身。
