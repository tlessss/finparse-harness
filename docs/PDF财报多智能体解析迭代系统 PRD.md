# PDF财报多智能体解析迭代系统 PRD

## 1\. 文档概述

### 1\.1 产品定位

本产品是一款**基于多智能体协作的非标PDF财报自动化解析、校验、自迭代系统**，核心解决上市公司年报/季报/研报等非标PDF文档的财务、人力、成本数据解析难题。通过多智能体分工协作，实现PDF数据抽取、向量比对校验、解析异常根因分析、解析器自动优化/新建的全闭环能力，同时配套可视化界面，支持全程流程追踪、人工复核与版本管理。

产品核心特性：**无固定模板适配、自动纠错迭代、全流程可视化、解析器可持续进化**，适配营收结构、成本构成、研发费用、员工数据等非标结构化数据提取场景。

### 1\.2 需求背景（更新）

目前系统已完成核心底座搭建，\*\*已部署成熟向量数据库，且沉淀了大量已精准解析的财报PDF样本数据\*\*，具备完善的财报版式、指标语义、标准数据案例存量资产。传统财报解析的核心痛点转变为：新增非标财报排版样式多变、指标命名不统一、新版式规则无法自适应，导致新增文档解析漏检、错检；同时存量优质样本资产未被充分利用，缺乏自动化校验、问题溯源与解析规则自迭代能力，人工修正成本高、新场景适配效率低。

基于现有存量数据资产，依托多智能体协作架构，构建「存量样本赋能解析、新增文档校验纠错、解析规则自动迭代」的闭环能力，最大化利用已有向量数据与解析案例，解决新增非标财报适配难、解析准确率迭代慢、流程不透明的核心问题，大幅降低人工运维成本。

### 1\.3 适用范围

适用文档：企业年报、季报、招股说明书、券商研报等结构化\+半结构化PDF文档

解析指标：营收结构、成本构成、研发费用明细、期间费用、员工人员结构、股权结构等企业经营数据

使用场景：批量财报数据结构化、财务数据合规校验、解析规则自动化迭代、数据复盘溯源

### 1\.4 开发环境

基于 **Cursor IDE** 全流程开发，依托AI辅助编码、自动补全、bug修复、代码重构，快速落地多智能体逻辑与可视化功能。

## 2\. 整体产品架构

### 2\.1 架构模式

采用 **LangGraph \+ CrewAI 多智能体流水线架构**，分层解耦，状态可追溯、流程可可视化、迭代可闭环

层级划分：任务调度层 → 解析执行层 → 智能校验层 → 优化迭代层 → 数据存储层 → 可视化交互层

### 2\.2 核心技术栈（适配Cursor开发）

- **多智能体框架**：LangGraph（流程编排、状态机、分支逻辑）\+ CrewAI（角色分工、Agent任务定义）

- **PDF解析引擎**：MinerU（布局检测、表格解析、OCR）\+ Camelot（精准财务表格提取）

- **向量检索RAG**：基于**系统现有成熟向量数据库**，存量覆盖海量财报版式向量、指标语义向量、标准解析案例向量；搭载bge\-large\-zh\-v1\.5中文嵌入模型，无需从零搭建向量底座

- **大模型底座**：DeepSeek\-V2 / 通义千问72B（本地私有化部署）/ GPT\-4o（备选）

- **后端服务**：Python \+ FastAPI（轻量接口、任务管理）

- **数据存储**：PostgreSQL（结构化数据、解析日志）\+ Redis（任务队列、缓存）

- **前端可视化**：React \+ Next\.js \+ React Flow（智能体流程可视化）\+ AG Grid（数据表格展示）

- **开发工具**：Cursor IDE（全栈开发、AI辅助调试、代码生成）

### 2\.3 核心角色（四大智能体\+基础引擎）

1. **调度智能体（可选默认开启）**：全局任务分发、流程流转控制、异常拦截、迭代次数管控、状态同步

2. **解析执行引擎**：非AI基础组件，执行预设解析器规则，完成PDF文本、表格、指标的原始抽取

3. **数据校验智能体（核心）**：对接向量库，完成相似度比对、数据逻辑校验、完整性校验，输出异常报告

4. **解析优化智能体（核心）**：异常根因诊断，决策「修改现有解析器」/「新建解析器」，输出规则/代码优化方案

## 3\. 核心业务流程（1:1匹配用户设计）

完整闭环流程：文档上传 → 调度分发 → 原始解析 → 向量校验 → 异常分流 → 优化迭代 → 二次校验 → 入库归档/人工兜底

### 3\.1 流程详细步骤

1. **任务初始化**：用户上传PDF，系统生成唯一任务ID，识别文档基础属性（页数、是否图片PDF、是否加密），前置过滤无效文档

2. **任务调度分发**：调度智能体将任务下发至解析引擎，启动规则化解析

3. **原始数据解析**：MinerU\+Camelot解析PDF，提取营收、成本、研发、员工等结构化数据，输出JSON中间结果，标记解析失败区块

4. **向量比对校验**：校验智能体调用**系统存量向量数据库**，调取历史已验证的优质财报样本、标准指标语义库、各类版式结构模板，将当前PDF切片、结构化解析数据向量化后进行多维比对，完成三重校验：
        语义一致性：指标名称、字段释义、行业通用口径匹配校验

5. 数据合理性：数值区间、分项求和、占比逻辑、财务勾稽关系合规校验

6. 内容完整性：对照存量标准指标清单，校验核心字段无漏提、错提、错配

7. **结果分支流转**

    - 校验通过：数据直接归档入库，流程结束

    - 校验异常：生成详细异常报告（异常位置、异常类型、向量差异、参考样本），流转至优化智能体

8. **解析规则迭代决策**：优化智能体诊断根因，输出二选一决策：
        

    - 局部异常（字段改名、小排版变动）：修改现有解析器正则/截取规则/关键词匹配规则

    - 全局异常（版式完全改版、表格结构重构）：新建专属解析器，绑定对应文档特征标签

9. **迭代重试**：自动更新解析规则/加载新解析器，重新解析当前PDF，再次进入校验流程，最大迭代3次

10. **兜底归档**：迭代成功则入库沉淀；迭代失败则流转人工复核，人工修正后同步更新向量库与解析规则

11. **样本沉淀与库体迭代**：所有新增文档的优质解析结果、异常修复案例、迭代后的解析规则，自动同步更新至**现有向量数据库与案例库**，持续丰富存量样本池，形成“解析\-校验\-迭代\-沉淀”的正向飞轮，不断提升系统适配能力与解析准确率

## 4\. 核心功能模块设计

### 4\.1 PDF文档管理模块

- 支持单文件/批量PDF上传、文档预览、任务状态查看（待执行/解析中/校验中/迭代中/已完成/异常）

- 文档基础信息记录：文件名、上传时间、文件大小、解析耗时、迭代次数

- 异常文档单独归档，支持二次重跑任务

### 4\.2 多智能体解析模块

- 规则化解析引擎：支持财务表格、文本指标、跨页表格、合并单元格解析

- 多指标定向抽取：营收结构、成本构成、研发费用、员工数据、期间费用等自定义字段

- 解析结果结构化输出：JSON、表格可视化展示，支持原文对应定位

### 4\.3 向量校验模块

- 存量数据复用：依托系统**已有的海量财报向量数据**，包含各类行业、各版本财报版式切片、标准化指标向量、历史正误解析案例，无需重复建库，直接赋能比对校验

- 多维比对：版式结构相似度、指标语义相似度、数据逻辑相似度

- 可视化校验结果：相似度分值、异常点位高亮、标准样本对照

- 异常类型标准化：漏提、错提、单位错误、结构识别失败、逻辑错误

### 4\.4 解析器自迭代模块

- 根因智能诊断：自动识别排版变更、字段别名、规则失效、全新版式等问题

- 双模式迭代：更新旧解析器 / 新建专属解析器

- 解析器版本管理：版本号、更新时间、适配场景、修改日志可追溯

- 迭代次数限制：最大3次迭代，防止死循环

### 4\.5 全流程可视化模块（核心需求）

前端可视化六大核心区域，全程流程透明可追溯：

1. **任务总览区**：任务队列、运行状态、迭代次数、完成率、异常率统计

2. **PDF预览区**：原文在线预览，异常位置精准高亮标注

3. **解析结果区**：结构化提取数据表格，支持与原文对照查看

4. **向量校验区**：相似度得分、异常报告、参考标准样本对比

5. **智能迭代区**：根因分析结论、解析器修改方案、代码/规则预览、决策结果展示

6. **日志溯源区**：每轮解析、校验、迭代操作的完整日志，支持回溯

额外支持：React Flow 动态展示多智能体协作流程图，实时刷新节点运行状态

### 4\.6 人工复核模块

- 拦截多次迭代失败的高难度异常任务

- 支持人工修正数据、标注异常原因、确认解析器优化方案

- 人工标注数据自动沉淀为训练样本，更新向量库与规则库

### 4\.7 数据归档与导出模块

- 结构化数据持久化入库，支持条件检索、历史查询

- 支持解析结果导出 Excel/JSON，满足下游数据分析需求

- 解析器版本、迭代日志、异常案例永久留存

## 5\. 核心规则与约束

### 5\.1 解析器迭代决策规则

**场景1：修改现有解析器（局部优化）**

- 指标名称别名变更（研发支出→技术开发费用）

- 局部表格行列微调、少量单元格合并

- 单条匹配规则失效，整体版式未变

**场景2：新建解析器（全局重构）**

- 文档整体排版、表格结构、指标排布完全改版

- 新增大量全新指标模块，旧规则完全无法适配

- 与**存量向量库中同类版式样本相似度低于60%**，判定为全新版式

### 5\.2 迭代终止规则

- 单次任务最大迭代次数：3次

- 迭代成功：校验无异常，数据合规完整，终止流程并归档

- 迭代失败：3次未修复，自动进入人工复核流程

### 5\.3 向量校验规则

- 版式相似度≥80%：判定为同类文档，沿用历史校验标准

- 语义相似度＜60%：判定为指标识别异常，触发纠错流程

- 数据逻辑错误（分项和≠总值、占比超100%）：强制标记异常

## 6\. 非功能需求

### 6\.1 性能需求

- 单份100页以内财报PDF解析耗时≤30s

- 向量检索响应耗时≤2s

- 多任务并行处理，支持10份PDF同时解析

### 6\.2 准确性需求

- 常规标准化财报数据解析准确率≥95%

- 异常识别覆盖率100%，无漏检严重数据错误

- 迭代后同版式文档解析准确率持续提升

### 6\.3 可扩展性需求

- 支持自定义新增解析指标、新增文档类型

- 智能体模块可插拔，可新增专项校验/优化Agent

- 解析器可批量管理、批量适配新场景
