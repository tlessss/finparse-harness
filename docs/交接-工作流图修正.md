## C. 自愈轨的真实流向（据此重连 LINKS）

```
死信/needs_repair → heal_revenue(自愈入口)
   → 选择即验证(select) 选最像母本 →三岔:
        ├ 母本exact      → 复用 → registry → 归档
        ├ 母本部分       → fork 改(gen)
        └ 都不行         → 新建(gen)
   gen → 沙箱(sandbox) 跑 → 打分器(score) 对golden → 版本闸(gate)
   gate ├ exact          → certify → registry(注册表) → 归档
        └ 非exact/撞墙K轮 → review(人审认证关: 看/改代码/重过闸/认证)
   review → (认证) → registry
```
要点：① 闸的终点是 **exact 否则转人工**（不是"比上一版好就收"）；② 人审是**撞天花板后的兜底**，不是每份必经；③ registry 认证后，**同版式后续直接复用**（回到 select 命中）。
