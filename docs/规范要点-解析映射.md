# 财报披露规范 → 解析逻辑映射

> 依据：《公开发行证券的公司信息披露内容与格式准则第2号——年度报告的内容与格式（2025版）》
> 原文：`llm-docs/附件1：……2025版.pdf`（59 页）
> 用途：把"准则规定的披露口径"翻译成解析锚点/表头白名单/校验口径，喂给 Phase 1.2（找页+认表+认列）与 Phase 2.0（rule-driven）。

---

## 〇、最重要的 5 个发现（颠覆现有解析假设）

1. **准则不规定表头字样**：MD&A（第三节）是散文式要求，"营业收入构成""分产品"这些**精确表头在准则正文里根本不出现**。
   → 这些表头来自**交易所模板/惯例**，不是强制字样。靠字面关键词找表天然脆弱，应改为**按语义**（收入+占比 / 收入+成本+毛利率）识别。

2. **营收是 4 个维度，不是 3 个**：准则第二十五条要求"分别按**行业、产品、地区、销售模式**说明营业收入构成"。
   → 我们现在只解析 行业/产品/地区，**漏了"销售模式"**这一维。

3. **存在两张不同的营收表（徐工踩坑的根因）**：
   - **(A) 营业收入构成表**：金额 + **占营业收入比重** → 各维度占比之和应 ≈ 100%（含"其他"行）。**这才是我们要的表，硬规则适用。**
   - **(B) 分行业/产品分项表**：营业收入 + 营业成本 + **毛利率** → 毛利率**不该求和**。
   → 徐工 000425 就是解析器挑了 (B) 表、又把**毛利率**当成占比 → 求和 64% 误判。**修复关键：认表/认列必须区分 (A) 和 (B)。**

4. **分项列示有 10% 门槛**：准则只要求"占营业收入或营业利润 **10% 以上**的行业/产品/地区/销售模式"分项列示其收入/成本/毛利率。
   → (B) 表可能**只列大项**，分项之和天然 < 100%，不能套"占比和=100%"硬规则；只有 (A) 构成表才有"其他"凑齐 100%。

5. **客户/供应商名称是"鼓励"非强制**：准则强制的是"**汇总比例**"（前5名客户销售额占比、前5名供应商采购额占比、关联方比例）；**分别披露名称和金额是"鼓励"**。
   → 很多公司只有汇总比例、没有明细名单是**合规的**，不应算"漏提"。（特例：单一客户>50%/新增/ST/贸易占比>10% 时才强制列名）

## 一、营收结构 revenue_breakdown
**条款**：第二十五条（一）1. 收入与成本
**规定**：按 **行业 / 产品 / 地区 / 销售模式** 披露营业收入构成；≥10% 的项分项列示 营业收入/营业成本/毛利率。
**解析指引**：
- 目标表 = **(A) 占营业收入比重表**；表头语义白名单：`占营业收入比重 / 营业收入比重 / 占比`。
- 识别维度锚点：`主营业务分行业 / 分产品 / 分地区 / 分销售模式`（新增"销售模式"）。
- **避坑**：若表头只有"营业收入/营业成本/毛利率"而无"占比"列 → 这是 (B) 表，**不要把毛利率当占比**；占比缺失就置空，别硬凑。
- 校验：仅对 (A) 表用"占比之和≈100%"；(B) 表改用"分项收入之和 ≤ 营业收入合计"。

## 二、成本构成 cost_breakdown
**条款**：第二十五条（一）1.（营业成本主要构成项目）
**规定**：披露营业成本主要构成项目（如 **原材料、人工工资、折旧、能源和动力**）在成本总额中的占比；商业秘密可只披露最主要单项。
**解析指引**：
- 两种成本表：① 按**性质**（原材料/人工/折旧/能源动力）② 按**行业/产品**（与营收 (B) 表同源）。
- 科目白名单：`原材料 / 人工 / 职工薪酬 / 折旧 / 能源 / 动力 / 制造费用`。
- 校验：占比之和 ≈ 100%（按性质表）。

## 三、研发 rnd_info
**条款**：第二十五条（一）3. 研发投入
**规定**：① 研发人员的**数量、占比、学历结构、年龄结构**；② **研发投入总额及占营业收入比重**；③ **研发投入资本化比重**。
**解析指引**：
- 我们现在抓的"研发费用明细"（职工薪酬/研发材料…）其实是**附注里的研发费用科目表**，与准则要求的"研发投入总额/资本化"是**两个口径**——注意别混。
- 可新增字段：研发投入总额、占营收比重、资本化比重、研发人员数量/占比。

## 四、员工 employees
**条款**：第三十四条
**规定**：母公司及主要子公司 **报告期末在职员工数量、专业构成、教育程度**、薪酬政策、培训计划、需承担费用的离退休人数；劳务外包较大的披露工时与报酬。
**解析指引**：
- 专业构成**标准类别（准则举例）**：`生产人员 / 销售人员 / 技术人员 / 财务人员 / 行政人员` → 直接做白名单。
- 校验：专业构成之和 = 教育程度之和 = 在职员工总数（现有硬规则正确）。

## 五、前五大客户/供应商 top_clients / top_suppliers
**条款**：第二十五条（一）1.（销售客户与供应商）
**规定（强制）**：汇总披露 前5名客户销售额占年度销售总额比例、前5名供应商采购额占采购总额比例、及各自关联方比例。
**规定（鼓励）**：分别披露 前5名客户/供应商 名称与金额、是否关联。
**强制列名的特例**：单一客户/供应商 >50%、存在新增、ST 风险警示、贸易业务收入占比 >10%。
**解析指引**：
- **必抓**：4 个汇总比例（客户占比/供应商占比/客户关联方占比/供应商关联方占比）。
- 明细名单**可能缺失且合规** → 缺名单不算异常；有则抽 名称+金额+占比+是否关联。

## 六、章节定位锚点（替代模糊关键词找页）

| 字段 | 准则章节 | 建议页定位锚点（小节标题/句式） |
|---|---|---|
| 营收/成本/客户供应商/研发 | 第三节 管理层讨论与分析·第二十五条 | "主要经营情况" / "收入与成本" / "研发投入" |
| 员工 | 第四节 公司治理·第三十四条 | "员工情况" / "专业构成" / "教育程度" |
| 研发费用科目明细 | 第十节 财务报告·附注 | "研发费用"（附注科目表） |

> 注意：营收/成本/客户/供应商/研发(投入) 在 **MD&A**，员工在 **公司治理**，研发费用科目明细在 **附注**。
> 现有营收解析器全文盲扫、不分章节，是挑错表的一大来源（应先锚定 MD&A "收入与成本"小节）。

## 七、对现有解析逻辑的直接修正项（喂给 Phase 1.2 / 2.0）

1. 营收**认表**：区分 (A) 占比构成表 与 (B) 收入/成本/毛利率表，只对 (A) 抽占比。
2. 营收**认列**：表头驱动，认"占营业收入比重"列；无该列则占比置空，**禁止把毛利率当占比**。
3. 营收**维度**：新增"销售模式"维度。
4. 营收**校验**：(A) 表占比和≈100%；(B) 表用"分项收入和 ≤ 营业收入合计"。
5. 客户/供应商：以"4 个汇总比例"为主目标；明细缺失不算漏。
6. 章节锚定：营收类先定位 MD&A"收入与成本"小节，员工定位"员工情况"小节。
7. 这些"锚点 + 表头白名单 + 标准科目"应外置为规则配置（Phase 2.0 的 rule schema 雏形）。
