## 0. 一句话任务 + 为什么（硬约束）

做一个**人能看见并控制这套自愈解析系统的控制/审核台**。

**为什么是硬前置**：这是个会**自己写解析器代码、自己认证、自己改代码**的自主系统。**正式大规模跑（5000+ 家）之前，这个前端必须就绪**——否则人看不见、控制不住，agent 失控。**前端 = 大规模跑的 gate，不是可选项。**

## 1. 系统是什么（一分钟）

来一份财报 PDF → 系统给它的"营收分项"找/写一个专用解析器：
- 选择即验证挑最像的**已认证解析器**：命中(exact)就**复用**；部分像就 **fork 改**；都不行就 LLM **从零新建**。
- 终点**恒=完全正确(exact)**；想尽办法仍不行 → **转人工**（绝不留半成品）。
- 每个解析结果都带**溯源**（每个数字来自 PDF 哪页哪格）。

你要做的就是让这一切**对人可见、可审、可控**。

## 2. 要建的三块

### A. 自愈活动 / 运行台（看见系统在干嘛）
- 列表：每份报告一行 → `{股票, 年份, 动作(复用/fork/新建/转人工), 解析器, 得分, 轮数, 状态}`。
- 可筛"转人工队列"。
- **控制**：暂停 / 停止跑批（按钮，先发请求占位，后端后接）。

### B. 人工 review 认证关（核心，§八）
左右对照 + 能改代码：
```
┌─ 解析结果(结构化, 可改数值) ─┬─ PDF原文(渲染页 + 溯源红框高亮) ─┐
│ 点某行 → 右边高亮那一格      │                                  │
├─ 解析器源码(LLM写的, 可编辑) ─────────────────────────────────┤
│ [💾 改代码并重过闸] → 显示新得分/是否exact                       │
└─ 动作: ✅批准认证  ✏️改数值  ❌打回重写 ────────────────────────┘
```
- **看到 LLM 写的解析器源码 + 直接改 + 一键重过闸**（拿到新分数）→ exact 即可批准认证。这是"撞天花板转人工后的最终修复兜底"，人改一次代码整类同版式都修好。

### C. 死信 / 转人工队列
- 转人工的报告列表 + 进入 B 处理。

## 4. 数据契约（新功能后端还没接 → 你先定 shape + mock，后端后接）

**红线：后端这些接口我（另一线）后面接。你先按下面 shape 建 UI + mock 数据跑通，别等后端。** 约定 shape：

```ts
// 自愈活动行
type HealRecord = {
  stock_code: string; year: number;
  action: "reuse" | "fork" | "new" | "escalate";
  parser_key: string | null; score: number; rounds: number;
  status: "certified" | "needs_human" | "running";
}
// 审核任务(扩展现有 /review/task)
type ReviewTask = {
  id: number; stock_code: string; year: number;
  result: Record<string, {name:string; revenue_yuan:number; ratio_pct:number}[]>; // industries/segments/regions
  provenance: Record<string, {page:number; bbox:[number,number,number,number]}>;  // "industries[0].revenue_yuan" -> 出处
  parser_code: string;       // LLM 写的解析器源码(可编辑)
  pdf_url: string;           // 用于渲染原文页
}
// 改代码重过闸(后端后接) POST /review/{id}/recode {code} -> {score:number, exact:boolean, mismatches:[...]}
```
真值参考：`goldset/revenue_golden.json`、`show_provenance` 渲染的红框图（`test_results/prov_*.png`）可作 mock 素材。
