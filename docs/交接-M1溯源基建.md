## 0. 一句话任务

让抽出来的每个表格单元格**带上它在 PDF 里的坐标**，并让解析结果**带上"每个数字来自哪一格"的溯源映射**——这样后面的人工 review 才能"点结果行 → 高亮 PDF 原文"。

## 1. 为什么要做（一句话动机）

整个系统要做"人对照 PDF 原文审解析结果"。前提是每个数字知道自己的出处 (页码 + 单元格坐标)。**没有溯源，人工 review = 在 300 页 PDF 里大海捞针，不可用。** 这是后续 M3 审核 UI、§七 裁判"对账/独立视野"的物理地基。

## 2. 当前代码真实状态（以代码为准）

**`src/parsers/infra/table_scanner.py: scan_pdf()`** 现在返回：
```python
[{"page": int, "table": [[cell_str, ...], ...], "text": str, "section": str}, ...]
#  有页码 ✅   2D 单元格字符串数组      无坐标 ❌
```
- 用的是 `page.extract_tables()` —— **这个 API 丢掉了 bbox 坐标**。
- 解析器（如 `src/parsers/revenue/default.py`）拿 `item["table"]` 这个 2D 数组认列、切行、取数，**输出里没有任何"这个数来自哪一格"的信息**。
- 当前**没有**统一的带溯源的 `parse()` 契约。先看 `src/parsers/base.py` 和 `revenue/default.py` 现有返回形状，再做最小侵入式扩展。

## 3. 目标

### 3.1 抽表层加坐标（核心）

把 `scan_pdf()` 从 `extract_tables()` 换成 `page.find_tables()`，让每个表带坐标。**关键用法**（pdfplumber）：
```python
for tbl in page.find_tables():
    grid = tbl.extract()                 # 和原来一样的 2D 字符串数组（保证不破坏现有解析器）
    cell_bbox = [[ (c.bbox if c else None) for c in row ] for row in tbl.rows]  # 同形状的坐标网格
    # tbl.bbox = 整张表的 (x0, y0, x1, y1)
```
输出形状**追加字段、不动老字段**（加法式改动，保证 30 个测试不退步）：
```python
{"page": int, "table": [[str]], "text": str, "section": str,
 "cell_bbox": [[ (x0,y0,x1,y1) | None ]],   # 与 table 同形状
 "table_bbox": (x0,y0,x1,y1)}
```
> ⚠️ 校验点：`find_tables().extract()` 的网格分割可能和 `extract_tables()` 不完全一致。**必须确认 `grid` 和 `cell_bbox` 行列对齐、且和老 `table` 兼容**（否则解析器认列会错位）。对不齐就以 `find_tables()` 的 grid 为准，同步更新 `table` 字段。

### 3.2 解析输出带溯源

解析器认到 name/revenue/ratio 列、取第 r 行的值时，把 `(page, cell_bbox[r][col])` 记进一个溯源映射，键为输出字段路径。目标契约（见设计 §六）：
```python
{
  "字段值": { "营收分项": [ {"name":..., "收入":..., "占比":...}, ... ] },
  "溯源":   { "营收分项[0].收入": {"page":87, "bbox":(x0,y0,x1,y1)},
             "营收分项[0].占比": {"page":87, "bbox":(...)}, ... }
}
```
先打通**营收**一个字段即可（最大瓶颈、验收最清楚），其它字段后续照搬。
