# FinParseAI 数据目标清单

> 核心目标：从非标 PDF 财报中提取结构化经营数据，存入 `financial_reports` 表，
> 供财务分析、数据校验、智能迭代使用。

---

## 一、总体概要

| 指标 | 数值 |
|------|------|
| `financial_reports` 总记录数 | 27,109 |
| PDF 缓存文件数 | 1,289 |
| PDF 已解析完成记录 | 部分（缺 revenue_breakdown 等），多数待解析 |
| 数据来源 | akshare（三张表自动采集）/ hybrid（PDF 解析覆盖） / pdf（纯 PDF） |

---

## 二、数据范围边界

| 数据来源 | 覆盖范围 |
|----------|---------|
| **akshare（已有，不处理）** | 三张主表（利润表 / 资产负债表 / 现金流量表）下的所有独立 DECIMAL 字段：`revenue`、`cost`、`selling_expense`、`admin_expense`、`rnd_expense`、`finance_cost`、`operating_profit`、`net_profit`、`total_assets`、`total_liabilities`、`cash`、`accounts_receivable`、`inventory`、`fixed_assets`、`operating_cf` 等 |
| **FinParseAI 从 PDF 解析（核心产出）** | 三张主表**附注**中的明细结构数据（JSON 字段）——营收结构、成本构成、研发费用明细、员工构成、前五大供应商/客户 |

> 三张主表的**汇总金额** akshare 已自动采集，FinParseAI 不做重复解析。
> FinParseAI 只处理 PDF 中独有的 **附注明细数据**。

### 核心 JSON 字段（FinParseAI 核心产出）

这是 FinParseAI **最主要的数据产品**，也是 PDF 独有的数据价值——akshare 无法提供这些结构化的经营数据。

#### revenue_breakdown（营收结构）

对应解析器：`RevenueParser`
代理指标：营业收入

```json
{
  "segments": [
    {"name": "互连类芯片", "revenue_wan": 154602.72, "ratio_pct": 86.05}
  ],
  "industries": [],
  "regions": [
    {"name": "境内", "revenue_wan": 165432.10},
    {"name": "境外", "revenue_wan": 25123.45}
  ]
}
```

| 子字段 | 含义 | 多氟多 PDF 中位置 |
|--------|------|------------------|
| `segments` | **分产品**（卖什么） | 主营业务分产品情况表 |
| `industries` | **分行业**（在哪些行业） | 主营业务分行业情况表 |
| `regions` | **分地区**（在哪些地方卖） | 主营业务分地区情况表 |
| 每项: `name` | 分类名称 | — |
| 每项: `revenue_wan` | 营收（万元） | — |
| 每项: `ratio_pct` | 占比（%） | — |

> 多氟多 2025 年报中预计出现在 103-104 页"营业收入"附注，含分产品/分地区毛利表。

#### cost_breakdown（成本构成）

对应解析器：`CostParser`
代理指标：营业成本

```json
[
  {"item": "原材料", "industry": "集成电路产品", "ratio_pct": 66.48, "amount": 136577.41},
  {"item": "人工", "industry": "集成电路产品", "ratio_pct": 12.35, "amount": 25360.00}
]
```

| 子字段 | 含义 | 说明 |
|--------|------|------|
| `item` | 成本项目名称 | 如"原材料"、"人工"、"制造费用"、"能源" |
| `industry` | 所属行业（可选） | 跨行业企业的成本分行业 |
| `ratio_pct` | 占比（%） | — |
| `amount` | 金额（万元） | — |

> 年报位置：附注"营业成本构成"。多氟多附注中与营收结构同页。

#### top_suppliers（前五大供应商）

对应解析器：`CostParser`
代理指标：营业成本

```json
{
  "total_amount": 322.1,
  "total_ratio_pct": 25.94,
  "items": [
    {"rank": 1, "name": "供应商A", "amount": 81.7, "ratio_pct": 6.58}
  ]
}
```

> 年报位置："主要销售客户和主要供应商情况"章节。
> 多氟多 2025 年报中预计在 57-58 页附近。

#### rnd_info（研发数据）

对应解析器：`RDParser`
代理指标：研发费用

```json
{
  "total_rnd": "915030216.00",
  "expensed_rnd": 915030216.00,
  "capitalized_rnd": 0,
  "rnd_staff": 583,
  "rnd_staff_ratio": 74.4,
  "rnd_salary": 245600000.00,
  "rnd_detail": [
    {"name": "职工薪酬", "amount_this": 120108558.42, "amount_last": 111687927.32},
    {"name": "研发材料", "amount_this": 254103216.55, "amount_last": 174797530.42}
  ]
}
```

> **多氟多 2025 测试验证结果**（第 205 页）：
> - `rnd_detail` 含 10 个明细项，合计 **487,287,219.43 元**（本期）
> - 完整结构：研发材料、职工薪酬、折旧费、动力费用、委外研究、差旅费、检验费、股份支付、其他
> - 上期合计 395,288,603.69 元，同比 +23.3%

#### employees（员工数据）

对应解析器：`EmployeeParser`
代理指标：员工总人数

```json
{
  "total": 8040,
  "parent": 2575,
  "composition": [
    {"type": "生产人员", "count": 5787},
    {"type": "技术人员", "count": 802},
    {"type": "销售人员", "count": 141},
    {"type": "财务人员", "count": 102},
    {"type": "行政人员", "count": 1208}
  ],
  "education": {
    "博士": 12,
    "硕士": 261,
    "本科": 2279,
    "大专及以下": 5488
  }
}
```

> **多氟多 2025 测试验证结果**（第 51 页）：
> - 总数: 8,040 人（母公司 2,575 + 子公司 5,465）
> - 专业构成：5 类
> - 教育程度：4 类
> - 无离退休职工

#### top_clients（前五大客户——待完善）

对应解析器：待新建
代理指标：营业收入

```json
{
  "total_amount": 321.5,
  "total_ratio_pct": 16.90,
  "related_party_ratio_pct": 2.83,
  "items": [
    {"rank": 1, "name": "客户A", "amount": 93.9, "ratio_pct": 4.94}
  ]
}
```

> 年报位置：与 top_suppliers 同表（主要客户和供应商情况）。
> agent-platform 尚未完善此解析器，FinParseAI 需自行实现。

## 三、数据落地流程

### 从 PDF 到 financial_reports 的映射

```
PDF 原始页面（附注章节）
    │
    ▼
FinParseAI 解析引擎（PyMuPDF + Camelot）
    │
    └── 表格 → Camelot → 结构化 JSON
              ├── 营业收入附注表 → revenue_breakdown（segments / industries / regions）
              ├── 成本构成附注表 → cost_breakdown
              ├── 研发费用附注表 → rnd_info（含 rnd_detail）
              ├── 员工构成表 → employees（composition + education）
              ├── 主要客户表 → top_clients
              └── 供应商表 → top_suppliers
    │
    ▼
update_report_fields(report_id, {
    "revenue_breakdown": {...},
    "cost_breakdown": [...],
    "employees": {...},
    "rnd_info": {...},
    "data_source": "hybrid",
    "pdf_parsed_at": "2026-06-09T12:00:00"
})

# 注意：不更新 revenue / cost / selling_expense 等独立字段
# 这些由 akshare 自动维护
```

## 五、解析优先级（按财务分析价值）

| 优先级 | 字段 | 分析价值 | 对应 PRD 阶段 |
|--------|------|---------|-------------|
| P0 | 研发费用明细（rnd_info.rnd_detail） | 判断"真研发 vs 假研发"——无形资产摊销占比 | 第一阶段 MVP |
| P0 | 员工数据（employees） | 判断人员结构合理性、人均薪酬竞争力 | 第一阶段 MVP |
| P0 | 营收结构（revenue_breakdown） | 判断增长战略（新赛道 vs 市场集中） | 第一阶段 MVP |
| P1 | 成本构成（cost_breakdown） | 判断成本转移能力（原材料占比 vs 毛利率） | 第一阶段 MVP |
| P1 | 前五大供应商（top_suppliers） | 供应商集中度/稳定性 | 第一阶段 MVP |
| P2 | 前五大客户（top_clients） | 客户集中度/关联交易 | 第二阶段 |
| P2 | 费用明细（销售/管理/财务） | 费用结构一致性分析 | 第二阶段 |

## 六、数据库表字段参考

所有字段定义见 `financial_reports` 表的 DDL。核心字段清单已在下文列出：

### 独立字段（DECIMAL，akshare 维护，FinParseAI 不处理）

> 以下字段由 akshare 自动采集，FinParseAI **不解析不写入**。
> 仅在 Agent 校验时读取用于交叉验证。

- `revenue` / `cost` / `selling_expense` / `admin_expense` / `rnd_expense` / `finance_cost`
- `operating_profit` / `profit_total` / `net_profit` / `parent_net_profit`
- `total_assets` / `total_liabilities` / `total_equity` / `parent_equity`
- `cash` / `accounts_receivable` / `inventory` / `fixed_assets` / `intangible_assets` / `goodwill`
- `short_term_loan` / `long_term_loan`
- `operating_cf` / `investing_cf` / `financing_cf` / `end_cash`
- 比率：`gross_margin` / `net_margin` / `roe` / `roa` / `debt_ratio` / `current_ratio`

### JSON 字段

| 字段 | 含义 | 解析器类型 | 优先级 |
|------|------|-----------|--------|
| `revenue_breakdown` | 营收结构（segments/industries/regions） | 生成 | P0 |
| `cost_breakdown` | 成本构成（item/industry/ratio_pct/amount） | 生成 | P1 |
| `employees` | 员工构成（total/composition/education） | 生成 | P0 |
| `rnd_info` | 研发数据（total_rnd/rnd_detail/rnd_staff） | 生成 | P0 |
| `top_suppliers` | 前五大供应商（items/ratio_pct） | 生成 | P1 |
| `top_clients` | 前五大客户（items/ratio_pct） | 生成 | P2 |
| `pdf_parse_flags` | 解析状态标记（{"rev":"ok","emp":"ok"}） | 自填 | P0 |
| `quality_flags` | 质量评分和问题清单 | 自填 | P0 |

### 状态字段

| 字段 | 含义 | 说明 |
|------|------|------|
| `data_source` | 数据来源 | `akshare` / `pdf` / `hybrid` |
| `pdf_parsed_at` | PDF 解析时间 | FinParseAI 解析完成时写入 |
| `quality_score` | 质量评分 | 0-1.00 |
| `ai_verdict` | AI 校验结论 | `pass` / `needs_fix` / `failed` |
