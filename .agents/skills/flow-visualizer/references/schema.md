# 数据结构参考

模板里你只编辑顶部这四个常量，其余是渲染引擎。

## META

```js
const META = { title: "标题", subtitle: "一句话副标题" };
```

## GROUPS — 分组（颜色 + 图例）

每个逻辑层一个分组。节点按 `group` 自动着色：描边用该色，填充自动半透明同色。

```js
const GROUPS = {
  myGroup: { color: "#1f6feb", label: "图例上显示的文字" },
};
```

建议每层一个颜色，颜色之间区分度高。参考配色：
`#1f6feb` 蓝 · `#0891b2` 青 · `#7c3aed` 紫 · `#b45309` 橙 · `#15803d` 绿 · `#8b5cf6` 浅紫。

## NODES — 节点

```js
{
  id: "唯一标识",         // 必填，edges 用它引用
  row: 0,                 // 必填，第几层（0 在最上，往下递增）；同 row 自动横向居中并排
  group: "myGroup",       // 必填，决定颜色
  label: "主标题",         // 必填，框内大字（保持短）
  sub: "副标题",           // 可选，框内小字
  planned: true,          // 可选，true=虚线框（标记"待开发/未实现"），默认实线
  w: 300,                 // 可选，框宽，默认 300。同一行总宽需 ≲ 860，否则会挤
  detail: {               // 可选，点击后右侧面板内容
    file: "real/path.py", // 真实文件路径
    what: "它做什么。",     // 一句话职责
    points: ["要点1", "要点2"], // 0–3 条
    data: "真实数字",       // 可选，量化信息（必须实测，别编）
  },
}
```

**布局要点**
- `row` 控制纵向分层，可以跳号（用 0,1,2… 或留空层）。
- 同 `row` 的多个节点会自动等距居中并排——用来表达"并列子步骤"或"工具集"。
- 一行节点总宽（含间距）建议不超过画布宽 860；超了就调小各自的 `w` 或拆到两行。

## EDGES — 边

```js
{ from: "id_a", to: "id_b", label: "可选文字", dashed: false }
```

**方向自动推断**（按两端节点的 `row`）：
- `to` 在 `from` 下方（row 更大）→ 向下箭头（曲线）。
- 同一 `row` → 横向直线。
- `to` 在 `from` 上方（row 更小）→ 自动走右侧绕回去 = **回环/反馈**。闭环、写回、循环都用这种，配 `dashed: true` 更清楚。

**表达 loop 的惯用法**：把循环体（如 4 个工具）放在中枢节点的下一行并列，再加一条从其中一个回中枢的 `dashed` 边，标注"结果回灌（循环）"。这比画 4 条回边干净。

## 不要碰的部分

`const W = 860 …` 往下是布局算法 + SVG 渲染 + 点击交互。换系统只改上面四个常量即可。
